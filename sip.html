Custom Abilities

Creator:
Jordan Hall, Game Programming
Last Updated: 1/7/2022

Technical Field

This project involves the technical field of programming.

Background Information

I got my inspiration to create this project by watching anime and wanting to see something I thought would be interesting to put in a game.

Prior Art (legal term)

My inspiration does not come from a game that already exists, but an anime called “Death Marth to the Parallel World Rhapsody”. The anime is in a game like theme where the player/Character goes through the world and learns things by experiencing them. The pro for to video game is that it gives players more of a sense that there is infinite number of things they can do. A con to a system like this is that it can be overwhelming at first from all the things that can be done.

Project Description

My project will be a system that changes abilities based on the player’s actions. There will be scenarios that player will end up doing, and each scenario can affect the stats on the abilities that the player uses. This will benefit the player because they can play the game in different ways than another player and different each time they play if played multiple times.

Innovation Claim

This project innovation is that the data on the abilities will be used to update how to they work later on.

Usage Scenario

For this project, the usage scenario is any game that uses abilities/spell that the developers want to change if the player does certain events and effects the stats. For example, if a player learned an ability related to ice and they did an event that used a similar ability but in a different way. The player’s character could learn the how to adapt its own ice abilities to be similar to the ice ability they come across when doing the event. With that the player can have different forms of the ice abilities they learn.

Evaluation Criteria

The following questions will identify the successful completion of the project.

  • Can player learn/upgrade their abilities through event and experiences?
  • Can designer easily make a new event for a chance to change ability stat.
  • Does the player have the ability to use the different style of abilities?

Objectives and Tasks Associated with the Project

Goal – Create a system that takes how an ability was used and changed it based on that experience.

  1. Objective 1 – Create a UI to show the ability stats.
    1. Show the current stats value and what they do in an easy to read form.
    2. Code the stat’s details to update the current stats.
    3. Design an ability the player can have.
  2. Objectives 2 – Design a case where the player ability can change.
    1. Code a system to take in data and figure out how to handle that data to update player's abilities.
    2. Create an scenario where the player's ability has a possiblity of changing.
    3. Link the system code to change player ability stats.
  3. Objective 3 - Create an ability system
    1. Design a UI to store abilities.
    2. Code ability to be able to be created by designers.
    3. Create a few test abilities to see the system working.

Description of Design Prototype

This prototype is built using Unity3d. This system is set up to allow designer to create events that the player can do if choice to do so. For this show of the system, you can move around and there are places you can go. Getting to the places may chance to stats of the ability you have shown on the screen. Depending the order in which you go to each place will determine what your stats will look like.

Evaluation Plan

In this prototype of the player learn their abilities by exploring and having their character learn the world around them. Since this was something small the exploring is not that far or that event full. That can all be changed by the designer to set up the world event that they want the player to follow to get to a certain point of progression. In each stage of progression, the player will still have the older version of their abilities and can use them whenever they like to.
